# Harness Principles

This skill uses a planner / drafter / reviewer structure because long-running research work is more reliable when generation and evaluation are separated and the intermediate state is saved in reusable artifacts.

## Core ideas

- Separate planning, drafting, and evaluation so the writing agent is not grading itself
- Use markdown artifacts as the working memory for a long run
- Review against concrete criteria instead of vague quality judgments
- Route failures precisely: rewrite, search again, or ask a human for help
- Pause when access limits are the true blocker instead of faking completeness

## Why this matters

Long runs tend to drift when:
- the context becomes long and the reasoning trail is not externalized
- the drafting agent becomes too lenient toward its own output
- missing information is treated like a writing problem instead of an access problem

Structured artifacts and an independent reviewer reduce those failure modes.
