# 搜索记录格式

向用户交付结果后，在当前工作目录的 `research/` 子目录中保存搜索记录为 `.md` 文件（如目录不存在则创建）。

## 文件命名

```
主题-YYMMDD.md
```

- `主题`：简洁的中文主题标签，无空格（需要时使用连字符），例如 `新能源汽车市场`、`AI芯片供应链`
- `YYMMDD`：搜索日期，例如 `260323`
- 示例：`新能源汽车市场-260323.md`

如果同一主题此前已搜索过，创建新的独立文件并在 `related` 字段中链接之前的文件 -- 不要覆盖。

## 文件结构

```markdown
---
topic: 主题
date: YYYY-MM-DD
queries:
  - "检索词1（中文）"
  - "English query 2 filetype:pdf"
scope: 地域 | 时间范围 | 来源偏好
related:
  - 主题-260310.md   # 同主题上次搜索，留空则删除此字段
---

## 来源列表

| 标题 | 机构 | 日期 | 类型 | 访问 | 链接 |
|------|------|------|------|------|------|
| 标题文字 | 机构名 | 2024-Q3 | consulting report | official PDF | [官方页](URL) · [PDF](URL) |

访问标签：`official PDF` / `official page` / `licensed/paywalled` / `secondary mirror`
类型标签：`consulting report` / `white paper` / `broker research` / `regulator/exchange` / `company IR` / `association/standard`

## 主要内容摘要

**强制要求：来源列表中的每一条信源都必须有对应的摘要块，不得遗漏任何一条。**

对于上述来源列表中的每个来源，编写一个摘要块：

### [标题](URL)

- **目录/结构**: 列出报告的主要章节或部分，例如 "第1章 市场规模 · 第2章 竞争格局 · 第3章 政策环境 · 附录 数据表"
- **核心内容**: 一句话说明这份资料解决了什么问题
- **检索提示**: 后续写报告时，哪个主题应查此文件的哪个章节，例如 "市场规模数据->第2章；政策梳理->第4章"

## 亮点数据

列出值得直接引用的具体数字、统计数据或发现。使用项目符号格式：

- **数据点**: 具体数字或结论 | 来源: [机构名](URL) | 页码/章节（如知道）

## 资料覆盖评估

- **偏向类型**: 本次搜索以哪类资料为主（例如 咨询报告为主，缺乏监管/官方数据）
- **明显缺口**: 哪个角度或数据点没有找到权威来源

## 缺口与受限来源

列出存在但无法完整访问的来源：

- [来源名称](landing page URL) | 原因: client-only / paywalled / no public PDF | 备注
```

## 保存规则

- 写入前务必创建 `research/` 目录（如不存在）。
- 不要对搜索返回的来源进行超出原文的修改或摘要 -- 记录是忠实的存档，而非编辑性改写。
- 省略无内容的章节（例如 如果没有缺口，则省略"缺口与受限来源"）。
- 保持文件自包含：未见过对话的读者应能理解搜索了什么、找到了什么。
